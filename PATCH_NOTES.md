# SchoolHub Patch Notes

## v1.1.0 - SchoolHub 관리자 구조 개선 업데이트

### 핵심 변경사항

SchoolHub v1.1.0에서는 여러 학교를 더 안정적으로 관리할 수 있도록 관리자 구조와 학교 관리 기능을 크게 개선했습니다.

기존에는 관리자 페이지가 전체 서비스 기준으로 동작했지만, 이제는 관리자가 먼저 학교를 선택한 뒤 해당 학교의 게시글, 댓글, 분실물, 공지사항을 관리하는 구조로 변경되었습니다.

---

### 학교 관리 기능 추가

- 관리자 페이지에서 학교 목록을 확인할 수 있도록 개선했습니다.
- 새로운 학교를 추가할 수 있습니다.
- 기존 학교의 이름, slug, 활성화 상태를 수정할 수 있습니다.
- 학교를 비활성화하면 일반 사용자는 해당 학교 페이지에 접근할 수 없습니다.
- 게시글, 댓글, 공지사항, 분실물 데이터가 없는 빈 학교는 삭제할 수 있도록 했습니다.
- 데이터가 남아 있는 학교는 실수로 삭제되지 않도록 삭제를 제한했습니다.
- 학교 목록과 학교 선택 화면이 사전순으로 정렬되도록 개선했습니다.

---

### 학교별 관리자 구조 적용

- 관리자 메인 화면을 학교 선택 방식으로 변경했습니다.
- 학교별 관리자 대시보드를 추가했습니다.
- 학교별로 아래 항목을 따로 관리할 수 있도록 변경했습니다.

  - 게시글 승인 관리
  - 댓글 승인 관리
  - 분실물 승인 관리
  - 공지사항 관리
  - 게시판 관리
  - 분실물 관리

---

### 학교별 공지사항 개선

- 학교별 공지사항 작성, 수정, 삭제 기능을 정리했습니다.
- 학교를 새로 생성하면 기본 공지사항이 자동으로 생성되도록 했습니다.
- 기본 공지사항은 `게시판 이용 안내` 제목으로 생성됩니다.
- 기본 공지사항은 자동으로 고정 상태로 생성됩니다.

---

### 게시글 / 댓글 승인 관리 개선

- 학교별 승인 대기 게시글 목록을 분리했습니다.
- 학교별 승인 대기 댓글 목록을 분리했습니다.
- 승인 대기 상세 페이지에서 승인 또는 삭제할 수 있도록 정리했습니다.
- 관리자 페이지의 뒤로가기 링크를 학교별 구조에 맞게 수정했습니다.

---

### 분실물 기능 개선

- 학교별 분실물 승인 관리 기능을 정리했습니다.
- 학교별 분실물 목록 관리 페이지를 개선했습니다.
- 관리자 페이지에서 분실물 상세 확인 및 삭제가 가능하도록 정리했습니다.
- 분실물 등록 시 닉네임을 제외한 모든 항목을 필수 입력으로 변경했습니다.
- 날짜 입력값이 잘못된 형식으로 저장되지 않도록 검증을 추가했습니다.
- 날짜는 4자리 연도 형식을 기준으로 검사하도록 개선했습니다.

---

### 비활성화 학교 접근 차단

- 비활성화된 학교는 일반 사용자 페이지에서 접근할 수 없도록 변경했습니다.
- 학교 메인, 게시판, 분실물 등 직접 URL로 접근하는 경우도 차단되도록 수정했습니다.
- 관리자는 비활성화된 학교도 계속 확인하고 수정할 수 있습니다.

---

### 관리자 페이지 UI / CSS 리팩토링

- 기존 `admin.css` 하나에 모여 있던 관리자 CSS를 기능별 파일로 분리했습니다.
- 관리자 CSS를 다음과 같은 구조로 정리했습니다.

```txt
public/css/admin.css
public/css/admin/common.css
public/css/admin/login.css
public/css/admin/dashboard.css
public/css/admin/schools.css
public/css/admin/notices.css
public/css/admin/suggestions.css
public/css/admin/reviews.css
public/css/admin/board.css
```

- 기존 EJS 파일의 CSS 연결 방식은 유지했습니다.
- 관리자 헤더, 푸터, 학교 관리, 대시보드, 승인 페이지, 게시판 관리 스타일을 더 관리하기 쉽게 정리했습니다.

---

### 데이터베이스 안정성 개선

- 학교 slug 변경 후 기본 학교가 중복 생성될 수 있는 문제를 방지했습니다.
- 기본 학교 생성 로직을 개선하여 기존 학교 데이터가 있으면 새 기본 학교를 만들지 않도록 변경했습니다.
- 학교 삭제 시 관련 데이터 존재 여부를 확인하도록 안전장치를 추가했습니다.

---

### 기타 수정사항

- 관리자 페이지의 일부 깨진 레이아웃을 수정했습니다.
- 승인 대기 게시글 페이지의 카드 스타일이 정상 적용되도록 수정했습니다.
- 분실물 관리 페이지에서 잘못된 `item` 참조로 인해 발생하던 오류를 수정했습니다.
- 학교별 관리자 페이지의 링크와 폼 경로를 학교 slug 기반으로 정리했습니다.

---

### 정리

v1.1.0은 SchoolHub가 여러 학교를 안정적으로 지원할 수 있도록 관리자 구조를 크게 개선한 업데이트입니다.

이번 버전부터 SchoolHub는 학교별 게시판, 공지사항, 분실물, 승인 관리를 분리해서 운영할 수 있습니다.

## 수정된 사항

- 분실물 상태값인 `찾는 중`, `찾음` 표시 제거
- 분실물 관리는 다른 게시물처럼 승인 중심 구조로 변경
- 관리자 하단 관리 페이지에서는 승인보다 삭제 관리 중심으로 변경
- 일반 사용자 화면에는 관리자 페이지 링크가 노출되지 않도록 유지

## 알려진 사항

- 관리자 세션은 현재 기본 MemoryStore를 사용합니다.
- v1.0에서는 단일 학교 운영을 기준으로 구성되어 있습니다.
- 추후 여러 학교 확장, 관리자 권한 세분화, 세션 저장소 개선 등을 진행할 수 있습니다.
